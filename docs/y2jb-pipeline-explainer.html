Delivery · YouTube is the drop zone

download0.dat splash.html
bootstrap

The PS5 YouTube app (Cobalt browser) can load local splash / cache content from download0. That’s how this project gets JS to run inside a first-party app process — not by attacking youtube.com’s servers.

  • Install a YouTube PKG version that matches your firmware range
  • Place the jailbreak package where Cobalt expects splash cache
  • Opening YouTube runs splash.html + chain scripts

Userland · V8 “The Hole”

main.js · trigger() · CVE-2021-38003 family
JS RCE seed

Cobalt’s JS engine is based on V8. A special internal value — The Hole — is smuggled into a Map via a Promise.any / thenable trick (trigger()).

Corrupting Map bookkeeping next to typed arrays produces an out-of-bounds view of nearby heap objects. That is the seed of browser-style memory control.

  • Leak object addresses (addrof)
  • Read / write 64-bit words in-process
  • Still sandboxed — only the YouTube process so far

Native glue · ROP + syscalls

main.js global.js misc.js
process power

Unstable heap R/W is upgraded to stable primitives, then used to find loaded modules: eboot, libc, Cobalt libs, and libkernel.

global.js holds ROP gadgets and syscall numbers. Once call() / syscall() work, JS can act like native code: open files, sockets, mmap, notify…

  • Fake objects + typed-array base pointer games
  • Module base discovery
  • Notifications, string helpers, kill-app utilities

03

Kernel · Lapse or P2JB

lapse.js ≤10.01 · p2jb.js ≤12.70 · kernel.js
jailbreak

Process sandbox is not enough. Firmware-specific kernel bugs turn syscall power into kernel read/write and privilege elevation.

main.js picks the path: Lapse (AIO / race lineage from PSFree) on older FW, P2JB (kqueueex / socket UAF style) on newer FW up to 12.70.

  • kernel.js — high-level kR/W helpers once primitives exist
  • Credential / jail checks get patched → “jailbroken”
  • Unsupported FW aborts cleanly

Post-JB · kexp + elfldr

aioshellcode.js kexp-*.bin elfldr-ps5-*.elf
loader up

After jailbreak, the chain maps kexp shellcode (RWX via jitshm / mmap) and boots elfldr — a local payload server.

Binaries are found next to the splash cache under the YouTube sandbox path. elfldr listens on localhost:9021 so only the console itself (by default) can feed payloads.

  • Thread spawn → run kexp with pipe / allproc context
  • elfldr ready for TCP payload streaming
  • Sets the stage for HEN / homebrew / Payload Manager

05

Autoload · ELFs & updates

autoload.js update.js icon_update.js
payloads

With the loader alive, autoload looks for ps5_autoloader on USB or /data, reads autoload.txt, and streams each ELF/bin to port 9021.

Optional steps: apply y2jb_update.zip, refresh icons. If no config exists, Payload Manager can launch as the friendly web UI for payloads.

  • Lines like !1000 = wait 1000 ms
  • Per-app folders: ps5_autoloader_PPSA01650
  • Same end state as classic “send payload to PS5” loaders

06

UI only · splash & progress

splash.html ui.js
cosmetics

The cyberpunk HUD is not part of the exploit. It only paints progress and logs. The chain talks through four globals:

  • autoloader_ui(version?) — show UI
  • updateProgress(percent, message)
  • uiLog(message, type?)
  • hideUI() — on hard errors

If ui.js is missing, main.js installs a tiny fallback DOM so the jailbreak still runs.

// mental model

YouTube is just a trusted app that embeds a vulnerable browser. Escape JS sandbox → control the process → escape the process via the kernel → run an ELF loader → load homebrew. Splash UI is the loading screen for that movie.
